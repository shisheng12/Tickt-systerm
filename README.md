# 客服工单系统 Demo

## 🌐 在线访问

**部署选项**：本系统支持一键部署到多个免费平台

### 快速部署（推荐 Vercel）
```bash
# 安装 Vercel CLI
npm install -g vercel

# 进入项目目录
cd ticket-system

# 一键部署
vercel --prod
```

部署完成后获得在线访问链接，例如：
- `https://ticket-system.vercel.app`
- `https://ticket-system.netlify.app`
- `https://ticket-system.pages.dev`

详细部署指南请查看 **[DEPLOYMENT_GUIDE.md](./DEPLOYMENT_GUIDE.md)**

### 投诉等级规则
基于投诉表样版.xlsx 的跟进要求表格：

| 等级 | 跟进频次 | 首响要求 | 提醒规则 |
|------|---------|---------|---------|
| 一般工单 | 至少3天1次 | 分派后4小时内触达 | 4小时未首响提醒；72小时未1次跟进提醒 |
| 紧急工单 | 至少1天1次 | 分派后2小时内触达 | 2小时未首响提醒；24小时未1次跟进提醒 |
| 加急工单 | 至少1天2次 | 分派后1小时内触达 | 1小时未首响提醒；24小时未2次跟进提醒 |
| 特急工单 | 至少一天2次 | 分派后30分钟内触达 | 15分钟未首响提醒；每12小时提醒 |

2. **👤 责任人负载 & 超时率**
   - 柱状图展示每位责任人的处理中工单数 + 超时工单数
   - 堆叠显示，红色超时部分一目了然
   - 按超时率降序排列，优先展示高风险责任人

3. **🔥 紧急待处理工单**
   - 表格展示高/紧急优先级且未完结的工单
   - 显示超时状态（已超时 N 天 / 今日到期 / 剩余 N 天）
   - 点击工单号跳转到工单列表

✅ **智能建议**
- 根据当前数据自动生成决策建议（Alert 卡片展示）

  - ⚠️ **警告**：超时工单过多、待处理积压、进单快于处理速度
  - ℹ️ **信息**：某责任人超时率过高，建议协调资源
  - ✅ **成功**：工单处理状况良好
