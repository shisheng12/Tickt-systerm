### 4. 统计服务封装（dashboardService.ts）

**核心函数**：`getDashboardStats(query: DashboardQuery): Promise<DashboardStats>`

**支持的筛选维度**：
```typescript
interface DashboardQuery {
  startDate?: string;  // ISO 日期
  endDate?: string;
  status?: TicketStatus;
  priority?: Priority;
  assigneeId?: string;
}
```

**返回的完整统计数据**：
```typescript
interface DashboardStats {
  metrics: MetricsData;                       // 6个指标
  statusDistribution: StatusDistribution[];   // 状态分布
  priorityDistribution: PriorityDistribution[]; // 优先级分布
  trendData: TrendData[];                     // 趋势（14天）
  channelDistribution: ChannelDistribution[]; // 渠道分布
  assigneeWorkload: AssigneeWorkload[];       // 责任人工作量
}
```

**关键算法**：
- 平均响应时间 = Σ(首次分配时间 - 创建时间) / 已分配工单数
- 趋势数据：循环近14天，按日期统计创建数和解决数
- 超时工单：dueAt < now 且状态 ≠ resolved/closed

### 5. 样式与交互（Dashboard.css）

  - lg (PC): 4列

## 技术亮点

1. **统计服务高内聚**：所有计算逻辑封装在 dashboardService，前端组件只负责渲染
2. **筛选联动流畅**：useEffect 监听筛选条件变化，自动触发 loadStats
3. **ECharts 配置标准化**：统一 title、tooltip、legend、grid 样式
4. **设计规范一致性**：所有颜色遵循设计规范文档
5. **响应式布局**：Grid 系统 xs/sm/lg 断点适配
6. **性能优化**：300ms 延迟模拟真实后端，避免过度请求
